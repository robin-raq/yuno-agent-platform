- **Fix ‚Äî fail CLOSED on ambiguous gate signal.** Surfaced by the haiku run: the executor's signal clamp fell back to `allowed[0]`, which for a Compliance gate (`[approve, reject]`) was **approve** ‚Üí a payout could be approved without screening if the model emitted no valid DECISION. - **Known limitation (follow-up):** `main.ts` binds `0.0.0.0`, so `/mcp/:agentId` (callable tools incl. the payout simulator) rides the public bind ‚Äî only the *connection* is loopback, not the *serve*.
