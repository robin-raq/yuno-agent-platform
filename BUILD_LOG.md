Real end-to-end `scripts/template-smoke.ts` runs `tpl-cbp` through real Goose: **all 5 tools fired ✅**, FX quote total $8,121.99, payout → `requires_approval` ($8k > $5k).
- **Finding — model tier is a correctness property, not just cost.** With **haiku** the 4-agent template never called its tools — it broke persona and confabulated ("the MCP call fails"); the chain degraded from the first agent's hesitation. **Sonnet** ran the whole template cleanly (all tools, correct routing). Decision: default `GOOSE_MODEL=claude-sonnet-4-6` (config + `.env.example`); haiku is fine for routing/restating but not multi-tool agent roles. Override via env.
- **Fix — fail CLOSED on ambiguous gate signal.** Surfaced by the haiku run: the executor's signal clamp fell back to `allowed[0]`, which for a Compliance gate (`[approve, reject]`) was **approve** → a payout could be approved without screening if the model emitted no valid DECISION. Extracted pure `clampSignal` (prefers `complete`, then `reject`, then first) so gates fail closed; the unscreened haiku run now *fails* (loop cap) instead of silently approving. +3 tests.
- **Known limitation (follow-up):** `main.ts` binds `0.0.0.0`, so `/mcp/:agentId` (callable tools incl. the payout simulator) rides the public bind — only the *connection* is loopback, not the *serve*. Mitigated by per-agent scoping + simulated payouts, but a real deploy needs the MCP surface on a loopback-only port or behind auth.
